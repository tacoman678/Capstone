Introduction:

The goal of my final project was to familiarize myself with aspects of software engineering through creating my own api and web design that I have never seen before, as well as creating something that can strengthen my resume and create a baseline for future projects beyond this one. So, to satisfy these standards I have set for myself, I decided to create a web application that collects a specified number (user input) of a desired product (user input) on amazon via web scraping and displays the product name, price, image, and a number (user input) of relevant reviews pertaining to each listing. To accomplish this project, I have broken down the process into seven steps which are listed in the procedure below:

Procedure:

  1. Choose a programming language to build the api and web framework that is compatible with that api for building the web application.
  2. Use a web scraping library from the language I chose to create an API that scrapes Amazon.com for the desired product and product aspects (title, price, image, and reviews) based on user input.
  3. Use the same technique developed in Step 2 to create an api for another website like Ebay.com
  4. Use the APIs created in steps 2 & 3 to extract the product name, price, image, and relevant reviews from the scraped product listings and create a database that is compatible with your chosen web framework to store those items.
  5. Create and design a webpage that can be used to display/showcase projects like this one. Then create a webpage that can be linked to from the main page that has a form in the web application that allows users to input the desired product and the number of products they want to see.
  6. Connect the form on the web application to the api I created by handling the form submission, validating the input, passing it to my api which will retrieve relevant product data from Amazon.com, process the data so that it is in a displayable format on a webpage, and finally return the results.
  7. Deploy the web application to a web hosting service of my choosing.
  8. Test the web application to ensure it is working as expected.

To create my webscraping API, the first step is to download all the python libraries I need listed in a.) which will require a virtual enviroment. To make a virtual enviroment in VS Code, I needed to run the following commands in the terminal. python -m venv venv to create it, and venv\Scripts\activate to activate it. Once activated I can pip install all the libraries that I will need. After everything I need has been imported, I made a method that returns the html code for a given product search on Amazon.com shown in b.). I did this by checking if the input was a string and formatting it correctly as a url ex:www.amazon.com/s?k=shirts+for+men, then setting up a headless Chrome browser with undetected_chromedriver and various options to disable features for faster browsing, navigating through the browser to the Amazon search results page for the given product, retrieves the page's HTML source and closes the browser, and finally create a BeautifulSoup object which can be easily parsed through with the retrieved HTML and returns it. Next I made methods simlilar to the one seen in c.) that takes an HMTL parse tree (soup) and searches through it for a specific html element. Each respective element was found using the dev tools on chrome by finding the element containing the text that I want (title, price, image, and links). After I made methods for each element that I need, I needed to make a way of aggregating all that data so it can be passed to my micro-framework using flask. I chose to use a pandas dataframe for this because it allowed me to append multiple arrays together while giving each array a variable name that can be easily accessed. The code for this step is outlined in d.).
